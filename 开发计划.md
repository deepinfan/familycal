# HomeCal 开发计划（MVP）

## 1. MVP 范围

- 角色登录（角色选择 + 密码）
- 任务列表（我的任务 / 共享任务）
- 新增任务（手动优先，自然语言解析次之）
- 周/月日历视图
- 文档列表与可见性控制
- 管理后台（角色管理 + 系统配置）

## 2. 里程碑

## M1：基础设施（1-2 天）

- 初始化 Next.js 15 + TypeScript + App Router
- 接入 Prisma + PostgreSQL
- 完成数据模型迁移与种子数据（admin + 初始角色）
- 搭建全局布局与路由守卫

## M2：认证与角色（1-2 天）

- 登录页角色网格
- JWT 登录与中间件校验
- 30 天保持登录
- `/admin` 鉴权

## M3：任务与日历（3-4 天）

- 任务 CRUD 与权限控制
- 完成/取消/延期及日志
- 周视图与月视图（含任务标记）
- 自然语言解析 API（先实现接口与降级）

## M4：文档模块（1-2 天）

- 文档 CRUD
- 文档可见范围管理（角色多选/所有人）
- Markdown 渲染与编辑

## M5：管理员系统配置（1 天）

- LLM 提供商切换
- API Key 加密存储
- 默认语言配置

## M6：PWA 与推送（1-2 天）

- next-pwa 离线缓存（静态资源 + 页面请求）
- 离线回退页（`/offline`）
- Web Push 订阅与触发

## 3. 接口优先级

- `POST /api/auth`：登录签发 JWT
- `GET/POST /api/events`：查询与创建任务
- `PATCH/DELETE /api/events/:id`：变更任务
- `GET/POST /api/documents`：文档查询与创建
- `PATCH/DELETE /api/documents/:id`：文档变更
- `POST /api/llm/parse`：自然语言转结构化
- `POST /api/push/subscribe`：推送订阅

## 4. 验收标准（MVP）

- 非 admin 无法访问 `/admin`
- 所有角色可看全部任务；仅创建者可改删任务
- 负责人可完成/取消/延期任务（延期必填原因）
- 手动任务创建可用；LLM 不可用时可回退手动
- 文档权限生效（无权限角色不可查看）
- 切换中英文后任务标题按语言显示

## 5. 风险与预案

- LLM 解析不稳定：强制确认层 + JSON 校验 + 手动降级
- 多语言数据不完整：允许单语存储并打标待补翻译
- Web Push 浏览器差异：先支持主流 Chromium，再兼容 Safari
