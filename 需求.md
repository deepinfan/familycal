# HomeCal — 家庭共享日历 PWA 设计规格

**日期：** 2026-03-19
**状态：** 待实施

---

## 1. 产品概述

HomeCal 是一个面向家庭的共享日历与任务管理 PWA，支持多角色登录、任务分配、日历视图、文档共享和自然语言输入。所有内容支持中英文双语存储与切换。

---

## 2. 用户与角色

### 角色体系

- 一个固定的 **admin** 管理员账号，通过后台管理所有家庭角色
- 家庭角色由 admin 创建，初始角色：爷爷、奶奶、外公、外婆、爸爸、妈妈、大菲佣、二菲佣
- 角色可增删，每个角色有：姓名、头像 emoji、角色 ID、密码
- Admin 可修改任意角色密码，也可修改自己的密码

### 登录方式

- 登录页展示所有家庭角色头像网格
- 自动记住上次选择的角色（localStorage）
- 输入密码登录，可勾选"保持登录 30 天"（写入 30 天有效期 JWT cookie）
- 30 天内再次访问自动跳过登录页

---

## 3. 核心功能模块

### 3.1 主界面

底部标签栏导航，三个页面：

| 标签 | 路由 | 说明 |
|------|------|------|
| 任务 | `/` | 默认落地页，个人 + 共享任务列表 |
| 日历 | `/calendar` | 周/月视图 |
| 文档 | `/documents` | 共享文档列表 |

管理员额外可访问 `/admin`。

---

### 3.2 任务视图

- 分两组展示：**我的任务**（当前角色为负责人）和**共享任务**（所有家庭成员均可查看全部任务）
- 每条任务显示：标题、时间、负责人、类型、状态（待办 / 已完成）
- 已完成任务置灰并加删除线
- 右上角"+ 新增"按钮，支持自然语言输入或手动填写

> **任务可见性规则**：所有任务对所有家庭成员可见（家庭协作场景，无需隐藏任务）。"我的任务"是当前角色为负责人的任务，"共享任务"是其他人负责的任务。

#### 任务操作权限

| 操作 | 权限 |
|------|------|
| 删除 / 修改 | 仅任务创建者 |
| 标记完成 | 任务负责人 |
| 延期（需填原因） | 任务负责人 |
| 取消 | 任务负责人 |

#### "所有人"负责人处理

负责人选"所有人"时，为当时所有角色各插入一条 `EventAssignee` 记录。后续新增角色不自动加入历史"所有人"任务。

---

### 3.3 日历视图

默认**周视图**，可切换为月视图（去掉天视图）。

#### 周视图

- 顶部显示本周 7 天，有任务的日期显示彩色圆点
- 今天高亮（蓝色圆圈）
- 默认展开**全周所有任务**，按日期分组列出
- 今天的任务组高亮显示（蓝色边框）
- 点击某天日期，页面滚动定位到该天任务组

#### 月视图

- 完整显示当月所有日期，周六日红色
- 有任务的日期显示彩色圆点（多个任务显示多个圆点，最多 3 个）
- 点击某天，在**该行下方**内联展开当天所有任务
- 再次点击收起

---

### 3.4 新增任务

支持两种输入方式：

#### 自然语言输入

1. 用户输入自然语言，如"本周三下午两点，妈妈去买生日蛋糕"
2. 调用 LLM API 解析，返回结构化 JSON：
   ```json
   {
     "title_zh": "买生日蛋糕",
     "title_en": "Buy birthday cake",
     "datetime": "2026-03-18T14:00:00",
     "assignee": "mama",
     "type": "购物"
   }
   ```
3. 展示解析结果供用户确认或修改
4. 确认后写入数据库

#### 手动填写

表单字段：标题、时间、类型（学习 / 玩耍 / 家务 / 购物 / 其他）、负责人（单选或"所有人"）

#### 翻译策略

- 主路径：LLM 解析时同步生成中英文
- Fallback：LLM 不可用时调用 Google Translate API 翻译
- 手动填写时同样走翻译流程（后台异步）
- 数据库同时存储 `title_zh` 和 `title_en`，前端根据界面语言选择显示

---

### 3.5 文档视图

- 列表展示当前角色有权限查看的文档
- 每条文档显示：标题、可见范围标签、创建人、创建日期
- 任何角色均可添加文档，添加时设置哪些角色可见（多选，或"所有人"）
- 文档创建者可修改内容、修改可见范围、删除文档

---

### 3.6 管理员后台 `/admin`

#### 角色管理

- 查看所有家庭角色列表
- 添加新角色（姓名、头像 emoji、初始密码）
- 编辑角色姓名和头像
- 修改任意角色密码（包括 admin 自己）
- 删除角色

#### 系统设置

- AI 模型选择（DeepSeek / Claude / GPT-4）
- AI API Key 配置（加密存储于数据库 `SystemConfig` 表，不以明文返回前端）
- 界面默认语言（中文 / English）

```prisma
model SystemConfig {
  key       String @id  // "llm_provider", "llm_api_key_encrypted", "default_lang"
  value     String
  updatedAt DateTime @updatedAt
}
```

> API Key 写入时使用服务端加密（AES-256），读取时解密后仅在服务端使用，不返回给前端。

#### 文档内容格式

文档内容为 **Markdown**，前端使用 `react-markdown` 渲染，编辑时提供简单 Markdown 编辑器。

## 4. 技术架构

### 技术栈

| 层 | 技术 |
|----|------|
| 框架 | Next.js 15 App Router + TypeScript |
| 数据库 | Vercel Postgres (PostgreSQL) |
| ORM | Prisma |
| 认证 | 自定义 JWT（角色名 + 密码，无邮箱） |
| PWA | next-pwa（支持离线缓存与离线回退页） |
| 推送通知 | Web Push API（VAPID）— 新任务创建、任务变更时通知负责人 |
| LLM | 可配置适配器（DeepSeek / Claude / GPT-4） |
| 翻译 Fallback | Google Translate API |
| 部署 | Vercel |

### 认证流程

1. 用户选择角色 + 输入密码
2. Server Action 验证密码（bcrypt）
3. 签发 JWT（payload 含 `roleId`、`roleKey`、`isAdmin`），写入 HttpOnly cookie
4. 勾选"保持登录"时 cookie 有效期 30 天，否则为 session cookie
5. 中间件校验 JWT，未登录重定向到 `/login`
6. `/admin` 路由额外校验 `isAdmin === true`，否则重定向到 `/`

#### LLM 解析错误处理

- LLM 返回无效 JSON 或网络超时：前端显示"解析失败，请手动填写"，自动切换到手动表单
- Google Translate 也不可用时：仅存储用户输入的语言版本，另一语言字段留空，后续可补填

## 5. 数据模型

```prisma
model Role {
  id            String   @id @default(cuid())
  name          String
  roleKey       String   @unique  // "mama", "baba", etc.
  passwordHash  String
  avatarEmoji   String
  isAdmin       Boolean  @default(false)
  createdAt     DateTime @default(now())

  createdEvents      Event[]            @relation("creator")
  assignedEvents     EventAssignee[]
  createdDocs        Document[]
  docVisibilities    DocumentVisibility[]
  eventLogs          EventLog[]
  pushSubscriptions  PushSubscription[]
}

model Event {
  id          String   @id @default(cuid())
  titleZh     String
  titleEn     String
  datetime    DateTime
  type        String   // 学习/玩耍/家务/购物/其他
  status      String   @default("pending") // pending/done/cancelled/extended
  creatorId   String
  creator     Role     @relation("creator", fields: [creatorId], references: [id])
  assignees   EventAssignee[]
  logs        EventLog[]
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt
}

model EventAssignee {
  eventId  String
  roleId   String
  event    Event  @relation(fields: [eventId], references: [id])
  role     Role   @relation(fields: [roleId], references: [id])
  @@id([eventId, roleId])
}

model EventLog {
  id          String   @id @default(cuid())
  eventId     String
  event       Event    @relation(fields: [eventId], references: [id])
  actorId     String
  actor       Role     @relation(fields: [actorId], references: [id])
  action      String   // extended/cancelled/done
  reason      String?
  newDatetime DateTime? // 延期时的新时间
  createdAt   DateTime @default(now())
}

model Document {
  id          String   @id @default(cuid())
  title       String
  content     String
  creatorId   String
  creator     Role     @relation(fields: [creatorId], references: [id])
  visibleTo   DocumentVisibility[]
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt
}

model DocumentVisibility {
  documentId String
  roleId     String
  document   Document @relation(fields: [documentId], references: [id])
  role       Role     @relation(fields: [roleId], references: [id])
  @@id([documentId, roleId])
}
```

```prisma
model PushSubscription {
  id        String   @id @default(cuid())
  roleId    String
  role      Role     @relation(fields: [roleId], references: [id])
  endpoint  String   @unique
  p256dh    String
  auth      String
  createdAt DateTime @default(now())
}
```

**推送触发时机：**
- 新任务创建时，通知所有负责人
- 任务被修改/取消/延期时，通知负责人
- 用户首次登录后提示授权推送通知

## 7. 关键设计决策

- **无邮箱认证**：家庭成员（含老人、家政）不依赖邮箱，纯角色名+密码
- **双语存储**：所有事项入库时同步存中英文，切换语言无需重新翻译
- **LLM 可插拔**：统一适配器接口，admin 后台可切换模型和配置 API Key
- **翻译 Fallback**：LLM 不可用时自动降级到 Google Translate，保证功能可用
- **PWA 离线缓存**：缓存核心静态资源与最近访问页面，断网时可访问离线回退页并继续查看已缓存内容
