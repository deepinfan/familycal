# 任务附件功能设计文档

## 概述

为任务添加附件功能，支持图片和文件上传，复用文档附件的实现模式。

## 目标

1. 创建任务时可以上传多个附件
2. 任务列表默认不加载附件信息
3. 展开任务时按需加载附件
4. 图片附件显示缩略图，支持 Lightbox 查看
5. 非图片附件显示下载链接

## 技术方案

### 数据库设计

**修改 Attachment 模型：**
- `documentId` 改为可选（nullable）
- 新增 `eventId` 字段（可选）
- 添加 Event 关系

**修改 Event 模型：**
- 添加 `attachments` 关系

### API 设计

**1. 任务创建 API（POST /api/events）**
- 添加 `attachments` 字段到 schema
- 创建任务时批量创建附件记录

**2. 任务附件查询 API（GET /api/events/[id]/attachments）**
- 按需加载任务的附件列表
- 返回完整的附件信息（包括缩略图）

**3. 复用现有上传 API（POST /api/upload）**
- 无需修改，已支持文件上传和缩略图生成

### 前端设计

**1. 任务创建表单**
- 添加文件上传组件（复用文档的实现）
- 支持多文件上传
- 显示已上传文件列表
- 支持删除已上传文件

**2. 任务卡片**
- 展开时异步加载附件
- 图片附件：显示缩略图，点击打开 Lightbox
- 非图片附件：显示文件名和大小，提供下载链接

**3. 类型定义**
- EventItem 添加 `attachments` 字段
- 复用 Attachment 类型定义

## 实现步骤

1. 数据库迁移：修改 Attachment 模型
2. 修改任务创建 API
3. 新增任务附件查询 API
4. 修改前端类型定义
5. 修改任务创建表单
6. 修改任务卡片显示逻辑

## 风险评估

- **低风险**：复用现有文档附件实现
- **数据库迁移**：需要添加可选字段，向后兼容
- **性能影响**：按需加载，不影响列表性能
